# Context 层备忘录

## 用途

在 AI agent 规划或编辑之前，让工作区变得可理解。本层定义如何查找、验证和使用上下文。

此备忘录是 context 协议。它不存储项目事实。稳定项目事实应放在 `docs/project-context.md`。

## 何时使用

在每个任务开始时、上下文重置后、进入代码库中不熟悉区域时，或现有笔记与当前文件冲突时使用本层。

## Agent 协议

1. 阅读 `docs/index.md`，定位代码、文档、工作流和验证命令。
2. 阅读 `docs/project-context.md`，获取稳定项目事实。
3. 当某个事实会影响任务时，检查当前项目文件；不要只依赖过期笔记。
4. 在提出改动前，识别与任务相关的模块、命令和约束。
5. 如果项目笔记与当前文件冲突，以当前文件作为事实来源，并在任务笔记中记录该冲突。

## 允许的行动

- 阅读项目地图、上下文笔记、源文件、配置文件和 package manifests。
- 搜索相关实现、测试、命令和参考资料。
- 将缺失或过期的上下文记录为任务本地笔记。
- 在验证后建议更新长期上下文。

## 禁止的行动

- 当可以检查文件时，不要猜测项目结构、技术栈、命令或所有权。
- 不要把旧笔记视为比当前源文件更权威。
- 仍在探索时，不要编辑无关的上下文文件。
- 不要把未验证假设写入长期项目上下文。
- 不要在此备忘录中复制项目事实；应将其放入 `docs/project-context.md`。

## 输出

- 与任务相关的项目上下文简述。
- 将指导工作的文件、模块或命令。
- 需要确认的上下文缺口或冲突。
- 如果发现新的已验证事实，提供给 Memory 层的候选稳定事实。
