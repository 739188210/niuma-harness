# Testing Rules

## Purpose

These rules define testing preferences for engineering work. They are a lightweight preference layer and do not replace the Observation evidence schema or completion evidence requirements.

## Test preference

For behavior changes, prefer focused automated tests that prove the intended behavior or reproduce the defect.

If automated tests are not practical, record the manual verification used instead and why.

## Test integrity

Do not weaken, skip, delete, or rebaseline tests to make work pass. Test-changing actions remain governed by Policy.

## Reporting

Report relevant test or verification commands through the Observation evidence schema, including skipped checks and remaining unknowns.
