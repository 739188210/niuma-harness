# 输出模板

所有结论都使用同一主模板，保留保持简单、局部抽取、最小模式方案的三档比较。可以压缩单元格内容，但不要为了模板补造不存在的证据。

## 设计结论主模板

```md
## 设计结论

### 观察证据
- <可定位的变化轴、边界泄漏、重复编排、状态复杂度、事件语义或验证困难>

### 已比较方案
| 方案 | 当前收益 | 额外代价 | 结论 |
|---|---|---|---|
| 保持简单 | … | … | 选择 / 不选：… |
| 局部抽取 | … | … | 选择 / 不选：… |
| 最小模式方案 | … | … | 选择 / 不选：… |

### 推荐
- 当前选择：<保持简单 / 局部抽取 / 模式名称>
- 选择原因：<为何当前选择胜过另外两档>
- 最小边界：<保持简单时写“无新增公共边界”；局部抽取时说明私有函数/模块；模式时说明职责、依赖方向和真实调用点>
- 复查条件：<什么新证据会改变本结论>

### 验证
- <共同的行为或边界验证>
- <按当前选择补充的专项验证>
```

## 专项验证提示

按最终选择补充必要项，不要求填写不适用模式的验证：

- **保持简单 / 局部抽取**：当前分支或模块行为、错误处理和复查触发条件。
- **Strategy**：两个真实实现的行为、选择逻辑、调用方不依赖具体实现。
- **Adapter**：数据与异常映射、业务层不依赖外部类型。
- **State / 显式状态机**：合法迁移、非法迁移拒绝、副作用、重复请求和并发冲突。
- **Factory**：创建分支、默认值、不变量和非法输入。
- **同步 Observer**：订阅通知、顺序、失败传播、同步完成和事务边界。
- **异步发布订阅**：提交后发布、投递、重试、消费者幂等、顺序、告警和失败归宿。
- **Facade**：上层行为、错误/事务语义、调用方依赖面收敛。

## 任务级取舍记录

任务记录可复用主模板中的“观察证据”“已比较方案”“当前选择”“复查条件”和“验证”。只有该选择影响跨模块协作、公共接口、长期依赖或难回滚方向时，才建议另行建立 ADR；局部任务取舍无需升级为长期决策记录。
